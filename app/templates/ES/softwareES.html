Estructura del servidor

El servidor se ha implementado en un microordenador o microcontrolador, dando la posibilidad de utilizar plataformas como Raspberry Pi, Arduino u otros dispositivos compatibles con el Socket. Proyecto IO. La estructura del servidor se basa en el archivo server. py, que es el enlace central conectado a la base de datos. Del mismo modo, client. py, una aplicación de Flask que también se integra con la base de datos, crea una página cliente dedicada usando Flask y el motor de plantillas. El proceso de generar una página cliente permite al cliente verla compartiendo un servidor local usando la aplicación ngrok.

Interfaz de cliente

En la interfaz del cliente, los usuarios pueden seleccionar comandos de la base de datos. Tras autenticarse, pasan los comandos al cliente. Para enviar comandos del cliente al servidor, los usuarios se conectan con un nombre de usuario y una contraseña. El servidor procesa estas órdenes utilizando los datos de los sensores y dispositivos de medición conectados. Los resultados se devuelven al servidor y luego al cliente. Este flujo de comunicación se basa en la tecnología Socket.IO para la interacción en tiempo real. Además, existe un servidor paralelo a nivel de cliente.
