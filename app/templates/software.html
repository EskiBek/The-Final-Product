Server structure

The server, implemented on a microcomputer or microcontroller, supports platforms like Raspberry Pi, Arduino, and other Socket.IO-compatible devices. Its structure relies on the server.py file, which acts as the central link to the database. Additionally, client.py, a Flask application, integrates with the database to create a dedicated client page using Flask and the template engine. This process enables clients to view the page by sharing a local server via the ngrok application.

Next

Clients interface

On the client interface, users can select commands from the database. After authentication, they pass commands to the client. To send commands from the client to the server, users log in with a login and password. The server processes these commands using data from connected sensors and measuring devices. Results are sent back to the server and then to the client. This communication flow relies on Socket.IO technology for real-time interaction. Additionally, there exists a parallel server at the client level.
